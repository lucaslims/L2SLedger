# ADR-020 — Clean Architecture e Domain-Driven Design no L2SLedger

## Status

Aprovado

## Contexto

O **L2SLedger** é um sistema financeiro com regras de domínio sensíveis (lançamentos, conciliações, fechamentos, auditoria e compliance). Para garantir evolução segura, testabilidade e isolamento de regras críticas, é necessário adotar uma arquitetura que seja:

* Independente de frameworks
* Altamente testável
* Manutenível ao longo do tempo
* Clara na separação de responsabilidades

---

## Decisão

Adotar **Clean Architecture** com princípios de **Domain-Driven Design (DDD)** como base estrutural do sistema.

---

## Camadas

### 1. Domain

Responsável por conter o **coração do sistema**:

* Entidades
* Value Objects
* Regras de negócio financeiras
* Invariantes de domínio
* Exceções de domínio

O domínio **não depende** de nenhuma outra camada.

---

### 2. Application

Responsável por:

* Casos de uso (Use Cases)
* Orquestração de regras de domínio
* Definição de contratos (interfaces)
* DTOs de entrada e saída

Não contém detalhes de infraestrutura.

---

### 3. Infrastructure

Responsável por:

* Persistência de dados
* Integrações externas (Firebase, banco, serviços)
* Implementações técnicas dos contratos da Application

---

### 4. API

Responsável por:

* Exposição de endpoints HTTP
* Controllers **thin** (sem lógica de negócio)
* Tradução de protocolos (HTTP → Application)

---

## Princípios Arquiteturais

* Dependências sempre apontam para dentro
* Domain não conhece Application, Infrastructure ou API
* Application define contratos
* Infrastructure implementa contratos
* Regras financeiras nunca residem em controllers

---

## Consequências

### Positivas

* Isolamento forte de regras financeiras
* Código altamente testável
* Evolução segura do sistema

### Negativas

* Maior volume inicial de código
* Curva de aprendizado para novos contribuidores

---

## Conclusão

A adoção de **Clean Architecture com DDD** fornece ao L2SLedger uma base sólida, previsível e segura, adequada à criticidade de um sistema financeiro.
Essa decisão alinha-se com as melhores práticas da indústria para sistemas complexos e críticos, garantindo que o L2SLedger possa evoluir de forma sustentável ao longo do tempo.
