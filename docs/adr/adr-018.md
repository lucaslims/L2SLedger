# ADR-018 — Criptografia e Proteção de Dados Sensíveis no L2SLedger

## Status

Aprovado

## Contexto

O **L2SLedger** manipula **dados financeiros e dados pessoais**, cuja exposição pode causar impactos severos de ordem legal, financeira e reputacional.

Além das exigências da **LGPD**, boas práticas de engenharia exigem a adoção explícita de **criptografia e proteção de dados sensíveis**.

---

## Decisão

Adotar uma política explícita de **criptografia obrigatória**, aplicada em trânsito e em repouso, para todos os dados sensíveis do sistema.

---

## Criptografia em Trânsito

* Todo tráfego externo deve utilizar **HTTPS (TLS)**
* Comunicação interna deve utilizar TLS quando possível

---

## Criptografia em Repouso

* Banco de dados deve utilizar criptografia em repouso
* Backups devem ser criptografados
* Volumes de armazenamento devem ser protegidos

---

## Dados Sensíveis

Consideram-se dados sensíveis:

* Dados financeiros (valores, saldos, históricos)
* Dados pessoais identificáveis
* Credenciais e secrets

---

## Gestão de Chaves

* Chaves devem ser armazenadas fora do código
* Rotação periódica quando aplicável
* Acesso restrito a processos autorizados

---

## Consequências

### Positivas

* Redução significativa de risco de vazamento
* Conformidade com LGPD
* Maior confiança do usuário

### Negativas

* Pequeno impacto de performance
* Maior complexidade operacional

---

## Conclusão

A criptografia sistemática de dados sensíveis garante que o L2SLedger opere com **alto nível de segurança**, atendendo requisitos legais e boas práticas de engenharia.

Adotar essas práticas é essencial para manter a integridade e confiança no sistema financeiro.
