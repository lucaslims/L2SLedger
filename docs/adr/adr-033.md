# ADR-033 — Uso da Oracle Cloud Infrastructure (OCI) como Infraestrutura Base do L2SLedger

## Status

Aprovado

## Contexto

O **L2SLedger** requer uma infraestrutura que ofereça:

* Controle total de recursos
* Custos previsíveis
* Isolamento de ambientes
* Compatibilidade com Docker

Foi avaliada a adoção de serviços totalmente gerenciados, porém estes introduzem maior custo e menor controle para o estágio atual do sistema.

---

## Decisão

Adotar a **Oracle Cloud Infrastructure (OCI)** como provedor de infraestrutura base do L2SLedger usando os recursos Always Free para minimizar custos iniciais.

---

## Arquitetura Base

* VM para aplicação (frontend + backend) - Arm64 2 OCPUs, 12 GB RAM
* VM separada para banco de dados - Arm64 2 OCPUs, 12 GB RAM
* Rede privada entre componentes
* Exposição externa mínima

---

## Princípios

* Infraestrutura previsível
* Controle total de rede e recursos
* Isolamento por ambiente
* Escala planejada, não automática

---

## Segurança

* Subnets públicas e privadas
* Firewall e regras explícitas
* Sem IP público para banco

---

## Consequências

### Positivas

* Custos controlados
* Arquitetura simples
* Independência de serviços proprietários

### Negativas

* Maior responsabilidade operacional

---

## Conclusão

A escolha da **OCI como infraestrutura base** oferece ao L2SLedger equilíbrio entre custo, controle e segurança, adequado ao estágio atual do produto.
