# ADR-017 — Exportação de Dados Financeiros (CSV / PDF / API) no L2SLedger

## Status

Aprovado

## Contexto

O **L2SLedger** permite a visualização e análise de dados financeiros por meio da interface web. Além disso, usuários podem necessitar **exportar dados financeiros** para fins de:

* Contabilidade
* Auditoria externa
* Relatórios gerenciais
* Integrações com sistemas terceiros

A exportação de dados financeiros envolve riscos relevantes de **exposição indevida**, **uso incorreto** e **violação de compliance**, exigindo regras claras.

---

## Decisão

Adotar uma política explícita e restritiva de **exportação de dados financeiros**, garantindo controle, rastreabilidade e segurança.

---

## Formatos Permitidos

* **CSV** — uso contábil e integração
* **PDF** — relatórios gerenciais
* **API** — integração controlada (quando aplicável)

Outros formatos são proibidos sem ADR específico.

---

## Regras de Exportação

* Exportações exigem **permissão explícita** (ver ADR-016)

* Exportações devem respeitar:

  * Escopo do usuário
  * Período financeiro
  * Restrições de acesso

* Exportações devem ser **auditadas** (ver ADR-014)

---

## Conteúdo Exportado

* Apenas dados necessários ao objetivo declarado
* Exclusão de dados sensíveis desnecessários
* Possibilidade de mascaramento quando aplicável

---

## Performance e Limites

* Exportações grandes devem ser:

  * Assíncronas
  * Limitadas por tamanho e período

---

## Consequências

### Positivas

* Redução de risco de vazamento
* Melhor controle sobre dados financeiros
* Suporte a auditorias externas

### Negativas

* Maior complexidade de implementação
* Limitações para usuários avançados

---

## Conclusão

A exportação controlada de dados financeiros garante que o L2SLedger ofereça flexibilidade aos usuários sem comprometer segurança, compliance e rastreabilidade.
Adotar essas práticas é essencial para manter a integridade e confiança no sistema financeiro.
