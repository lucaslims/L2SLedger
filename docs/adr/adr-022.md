# ADR-022 — Contratos Públicos Imutáveis da API no L2SLedger

## Status

Aprovado

## Contexto

No **L2SLedger**, a API é consumida diretamente pelo frontend SPA. Mudanças não controladas nos contratos públicos podem causar:

* Quebras silenciosas no frontend
* Regressões funcionais
* Dificuldade de versionamento
* Risco operacional em ambiente financeiro

---

## Decisão

Definir que **todos os contratos públicos da API do L2SLedger são imutáveis**.

---

## Princípios

* Todo endpoint exposto é considerado **contrato público**
* Contratos não são alterados sem:

  * Novo ADR
  * Versionamento explícito
  * Atualização coordenada do frontend

---

## Escopo

Incluem-se como contratos públicos:

* Endpoints HTTP
* Estruturas de request e response
* Códigos de erro semânticos
* Comportamentos documentados

---

## Versionamento

* Alterações incompatíveis exigem nova versão de endpoint
* Versionamento deve ser explícito e documentado

---

## Sub-ADRs

Este ADR poderá ser refinado por **Sub-ADRs**, seguindo a convenção:

* `ADR-022-a.md`
* `ADR-022-b.md`

Cada Sub-ADR deve tratar um refinamento específico (ex: contratos financeiros, contratos de importação, contratos de auditoria).

---

## Consequências

### Positivas

* Estabilidade de integração
* Governança clara de mudanças
* Base sólida para agentes de IA

### Negativas

* Evolução exige planejamento e disciplina

---

## Conclusão

A imutabilidade dos contratos públicos garante que o L2SLedger evolua de forma **segura, previsível e auditável**, alinhada às exigências de um sistema financeiro.
