# ADR-035 — Estratégia de Migrations e Versionamento de Banco de Dados no L2SLedger

## Status

Aprovado

## Contexto

O L2SLedger evolui continuamente, exigindo alterações controladas no schema do banco de dados. Mudanças manuais ou não versionadas representam alto risco para dados financeiros.

---

## Decisão

Adotar uma **estratégia obrigatória de migrations versionadas** para o banco de dados.

---

## Princípios

* Toda alteração de schema é versionada
* Migrations são imutáveis após aplicadas
* Migrations fazem parte do código-fonte
* O mesmo pipeline aplica migrations em todos os ambientes

---

## Execução

* Migrations executadas via pipeline CI/CD
* Execução explícita e auditável
* Rollback apenas via migration compensatória

---

## Proibições

* Alterações manuais em produção
* Execução de SQL ad-hoc sem versionamento

---

## Auditoria

* Versão do schema registrada no banco
* Execuções registradas por ambiente

---

## Consequências

### Positivas

* Evolução segura
* Rastreabilidade completa
* Menor risco operacional

### Negativas

* Disciplina adicional no desenvolvimento

---

## Conclusão

A estratégia formal de **migrations e versionamento de banco** garante evolução controlada e segura dos dados financeiros do L2SLedger.
Implementar essa abordagem é crucial para manter a integridade e confiabilidade do sistema financeiro.
Adotar essa estratégia fortalecerá a governança de dados e facilitará futuras manutenções e auditorias.
