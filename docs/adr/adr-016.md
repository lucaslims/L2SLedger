# ADR-016 — Controle de Acesso e Permissões Financeiras (RBAC/ABAC) no L2SLedger

## Status

Aprovado

## Contexto

O **L2SLedger** manipula dados financeiros sensíveis e permite operações com diferentes níveis de impacto (visualização, lançamento, ajuste, fechamento). É necessário definir um modelo claro e extensível de **controle de acesso**.

---

## Decisão

Adotar um modelo híbrido de **RBAC (Role-Based Access Control)** com suporte a **ABAC (Attribute-Based Access Control)** para decisões contextuais.

---

## RBAC — Papéis Básicos

Papéis iniciais:

* **Admin**: controle total
* **Financeiro**: lançamentos, ajustes e conciliação
* **Leitura**: acesso apenas para visualização

Papéis podem evoluir conforme necessidade.

---

## ABAC — Atributos Contextuais

Decisões podem considerar:

* Usuário
* Período financeiro (aberto/fechado)
* Tipo de operação
* Origem da ação (manual/importação)

---

## Princípios

* Autorização é sempre decidida no backend
* Frontend apenas reflete permissões
* Permissões são avaliadas **a cada requisição**

---

## Integração com Auditoria

* Toda decisão sensível deve ser auditável
* Tentativas de acesso negado devem ser registradas

---

## Consequências

### Positivas

* Controle granular de acesso
* Menor risco de erro operacional
* Base sólida para crescimento do sistema

### Negativas

* Maior complexidade inicial

---

## Conclusão

O modelo RBAC/ABAC fornece ao L2SLedger um equilíbrio entre **simplicidade inicial** e **flexibilidade futura**, garantindo segurança e controle adequados para um sistema financeiro.
