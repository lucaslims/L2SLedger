# ADR-034 — Banco Relacional (PostgreSQL) como Fonte Única de Persistência no L2SLedger

## Status

Aprovado

## Contexto

O **L2SLedger** é um sistema financeiro com requisitos fortes de:

* Consistência transacional
* Integridade referencial
* Auditoria e rastreabilidade
* Relatórios financeiros confiáveis

Foi necessário decidir o modelo de persistência principal, considerando alternativas como NoSQL, bancos híbridos ou múltiplas fontes de dados.

---

## Decisão

Adotar **PostgreSQL como banco relacional único e oficial** para persistência dos dados do L2SLedger.

---

## Justificativa

* Suporte completo a transações ACID
* Forte consistência
* Constraints e integridade no nível do banco
* Suporte maduro a auditoria, views e relatórios

---

## Princípios

* PostgreSQL é a **fonte da verdade** dos dados financeiros
* Nenhum outro datastore é usado como fonte primária
* Caches, quando existirem, são apenas derivados

---

## Escopo

Persistidos no PostgreSQL:

* Lançamentos financeiros
* Categorias
* Saldos
* Auditoria
* Usuários internos do domínio

---

## Consequências

### Positivas

* Dados confiáveis
* Relatórios consistentes
* Base sólida para auditoria

### Negativas

* Escalabilidade horizontal mais complexa

---

## Conclusão

O uso de **PostgreSQL como banco relacional único** garante integridade, confiabilidade e previsibilidade para o L2SLedger.
