# ADR-032 — Docker como Padrão de Containerização no L2SLedger

## Status

Aprovado

## Contexto

O **L2SLedger** é executado em VMs na OCI e precisa garantir **portabilidade**, **consistência entre ambientes** e **facilidade de deploy**. Executar aplicações diretamente no host gera divergências e riscos operacionais.

---

## Decisão

Adotar **Docker como padrão obrigatório de containerização** para frontend e backend do L2SLedger.

---

## Escopo

* Frontend empacotado em container
* Backend empacotado em container
* Dependências isoladas por container

---

## Princípios

* Uma imagem por serviço
* Imagens imutáveis
* Configuração externa via variáveis de ambiente
* Containers stateless

---

## Build e Execução

* Build de imagens realizado no CI/CD
* Execução via Docker Compose ou equivalente
* Nenhum build em produção

---

## Segurança

* Imagens base oficiais
* Atualização periódica de imagens
* Scan de vulnerabilidades quando possível

---

## Consequências

### Positivas

* Portabilidade total
* Ambientes idênticos
* Deploy simplificado

### Negativas

* Overhead mínimo de containers

---

## Conclusão

O uso obrigatório de **Docker** garante padronização, isolamento e confiabilidade operacional no L2SLedger.
