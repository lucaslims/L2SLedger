# ADR-012 — Backups e Disaster Recovery (DR) no L2SLedger

## Status

Aprovado

## Contexto

O **L2SLedger** manipula **dados financeiros sensíveis**, cuja perda ou corrupção pode gerar impactos operacionais, legais e reputacionais.

O sistema utiliza:

* Backend .NET 10
* PostgreSQL em VM dedicada
* Containers Docker
* OCI como cloud provider

É necessário definir uma estratégia clara de **backup e recuperação de desastres (Disaster Recovery — DR)**, compatível com o porte atual do sistema, mas preparada para evolução futura.

---

## Decisão

Adotar uma estratégia de **backup obrigatório e DR planejado**, baseada em simplicidade, confiabilidade e testes periódicos.

---

### 1. Backups de Banco de Dados

* Backups automáticos e periódicos do PostgreSQL

* Frequência mínima:

  * Backup diário completo
  * Retenção mínima de 30 dias

* Backups armazenados fora da VM de dados

* Backups criptografados em repouso

---

### 2. Backups de Configuração

* Versionamento de:

  * Docker Compose
  * Scripts de infraestrutura
  * Configurações não sensíveis

* Segredos **não** incluídos em backups de código

---

### 3. Estratégia de Disaster Recovery

* Capacidade de restaurar:

  * Banco de dados
  * Aplicação
  * Infraestrutura básica

* RTO (Recovery Time Objective): **definido por ambiente**

* RPO (Recovery Point Objective): **máximo de 24 horas** no MVP

---

### 4. Testes de Recuperação

* Testes periódicos de restore
* Validação da integridade dos dados restaurados
* Registro de resultados

---

## Consequências

### Positivas

* Redução significativa de risco de perda de dados
* Maior confiança operacional
* Base sólida para compliance futuro

### Negativas

* Custo adicional de armazenamento
* Necessidade de disciplina operacional

---

## Alternativas Consideradas

* Ausência de política formal de backup (rejeitado)
* Backups manuais (rejeitado por risco humano)

---

## Conclusão

A definição explícita de **Backups e Disaster Recovery** garante que o L2SLedger possa se recuperar de falhas graves, protegendo dados financeiros críticos e assegurando continuidade operacional.
