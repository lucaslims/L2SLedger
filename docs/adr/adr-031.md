# ADR-031 — GitHub Actions como Pipeline de CI/CD no L2SLedger

## Status

Aprovado

## Contexto

O **L2SLedger** possui frontend SPA (React), backend (.NET 10), containers Docker e múltiplos ambientes (DEV, DEMO, PROD). Para garantir **qualidade, rastreabilidade e repetibilidade** dos deploys, é necessário definir uma estratégia clara de **Integração Contínua e Entrega Contínua (CI/CD)**.

---

## Decisão

Adotar o **GitHub Actions** como ferramenta oficial de CI/CD para todo o ciclo de vida do L2SLedger.

---

## Escopo do CI/CD

O pipeline deve cobrir:

* Build do frontend
* Build do backend
* Execução de testes automatizados
* Build de imagens Docker
* Publicação de artefatos
* Deploy controlado por ambiente

---

## Princípios

* Pipeline como código (versionado no repositório)
* Falha em qualquer etapa bloqueia o deploy (fail-fast)
* Mesmo pipeline para todos os ambientes
* Configuração sensível via secrets do GitHub

---

## Ambientes

* Branches controlam o fluxo:

  * `main` → PROD
  * `release/*` → DEMO
  * `develop` → DEV 

* Deploys exigem aprovação manual quando aplicável

---

## Segurança

* Segredos armazenados apenas no GitHub Secrets
* Nenhum segredo versionado no código
* Acesso mínimo necessário aos runners

---

## Consequências

### Positivas

* Automação confiável
* Redução de erros manuais
* Auditoria de deploys

### Negativas

* Dependência do GitHub

---

## Conclusão

O uso do **GitHub Actions como CI/CD** garante automação, previsibilidade e segurança no ciclo de entrega do L2SLedger.
