# ADR-037 — Estratégia de Testes Automatizados no L2SLedger

## Status

Aprovado

## Contexto

O **L2SLedger** é um sistema financeiro crítico, onde falhas podem gerar impactos legais, operacionais e de confiança. Para garantir previsibilidade e segurança, é necessário definir uma **estratégia formal de testes automatizados**, integrada ao ciclo de desenvolvimento e ao CI/CD.

---

## Decisão

Adotar uma **estratégia de testes automatizados em múltiplos níveis**, cobrindo domínio, aplicação, infraestrutura e contratos públicos.

---

## Pirâmide de Testes

1. **Testes Unitários**

   * Regras de negócio
   * Entidades e Value Objects
   * Casos de uso

2. **Testes de Integração**

   * Persistência (PostgreSQL)
   * Integrações externas (Firebase — mockado)

3. **Testes de Contrato**

   * Contratos públicos da API
   * Códigos de erro e semântica

---

## Princípios

* Backend é a fonte da verdade
* Testes devem ser determinísticos
* Falhas devem ser explícitas (fail-fast)
* Testes fazem parte do pipeline CI/CD

---

## Execução

* Testes executados a cada pull request
* Falha em testes bloqueia merge
* Cobertura mínima exigida por módulo

---

## Consequências

### Positivas

* Redução de regressões
* Confiança na evolução do sistema
* Base sólida para auditoria

### Negativas

* Tempo adicional de escrita de testes

---

## Conclusão

A estratégia formal de **testes automatizados** garante qualidade contínua e segurança no desenvolvimento do L2SLedger.
