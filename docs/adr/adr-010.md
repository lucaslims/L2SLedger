# ADR-010 — Política de Variáveis de Ambiente no L2SLedger

## Status

Aprovado

## Contexto

O **L2SLedger** utiliza múltiplos containers Docker e ambientes distintos (DEV, HML, PROD). Sem uma política clara de variáveis de ambiente, surgem riscos de inconsistência, falhas em produção e dificuldades de reprodução de ambientes.

---

## Decisão

Estabelecer uma **política única, obrigatória e padronizada** para definição, nomenclatura e uso de variáveis de ambiente em todo o ecossistema do L2SLedger.

---

## Regras de Nomenclatura

* Variáveis devem ser escritas em **UPPER_SNAKE_CASE**
* Prefixos por responsabilidade

### Prefixos Padrão

| Prefixo | Uso                               |
| ------- | --------------------------------- |
| `APP_`  | Configurações gerais da aplicação |
| `AUTH_` | Autenticação / Firebase           |
| `DB_`   | Banco de dados                    |
| `LOG_`  | Logging                           |
| `OBS_`  | Observabilidade                   |
| `SEC_`  | Segurança                         |

Exemplo:

```env
APP_ENV=production
DB_POSTGRES_CONNECTION_STRING=...
AUTH_FIREBASE_PROJECT_ID=...
```

---

## Escopo das Variáveis

### Obrigatórias

* Connection strings
* Secrets de autenticação
* Identificadores de projeto

A ausência deve resultar em **falha imediata de startup**.

---

### Opcionais

* Flags de feature
* Configurações de timeout

Devem possuir default seguro.

---

## Versionamento e Documentação

* Todo serviço deve possuir:

  * `.env.example`
  * Documentação das variáveis esperadas
* Arquivos `.env` reais **nunca** são versionados

---

## Comportamento em Runtime

* Variáveis são lidas apenas na inicialização
* Mudanças exigem restart do container
* Não permitir reload silencioso de segredos

---

## Consequências

### Positivas

* Ambientes previsíveis
* Onboarding facilitado
* Menor risco operacional

### Negativas

* Maior rigor na configuração inicial

---

## Conclusão

Uma política explícita de variáveis de ambiente é essencial para a **segurança, previsibilidade e operação confiável** do L2SLedger em ambientes Docker e cloud.
