# ADR-008 — Segurança de Rede (OCI) no L2SLedger

## Status

Aprovado

## Contexto

O **L2SLedger** opera em uma arquitetura baseada em **VMs na Oracle Cloud Infrastructure (OCI)**, utilizando **containers Docker**, com:

* **Frontend SPA (React)** e **Backend (.NET 10)** executando na **mesma VM**
* Apenas o **frontend exposto externamente**
* Backend não exposto publicamente
* Banco de dados PostgreSQL hospedado em **VM separada**, em rede privada
* Comunicação com serviços externos (Firebase Authentication)

Neste cenário, a **segurança de rede** é fundamental para reduzir a superfície de ataque, garantir isolamento entre camadas e prevenir acessos não autorizados.

---

## Decisão

### 1. Segmentação de Rede

* As VMs na OCI devem ser organizadas em **subnets distintas**:

  * **Subnet pública**: exposição controlada apenas do frontend
  * **Subnet privada**: backend e camada de dados

* O acesso direto ao backend a partir da internet é **explicitamente proibido**

---

### 2. Controle de Acesso

* Comunicação entre frontend e backend ocorre na **mesma origem**, internamente à VM
* Comunicação entre backend e banco de dados ocorre **exclusivamente via rede privada**
* Security Lists / NSGs devem permitir apenas:

  * Portas estritamente necessárias
  * Origem e destino conhecidos

---

### 3. Exposição Externa

* Apenas o **frontend** possui exposição externa
* Backend não possui IP público
* VM de dados não possui acesso direto à internet

---

### 4. Comunicação Segura

* Todo tráfego externo deve utilizar **HTTPS**
* Certificados TLS são gerenciados no nível de entrada (frontend / proxy)
* Comunicação interna pode utilizar TLS conforme necessidade

---

### 5. Princípios de Segurança

* Princípio do **menor privilégio**
* Isolamento de camadas
* Nenhuma confiança implícita baseada apenas em localização de rede

---

## Consequências

### Positivas

* Redução significativa da superfície de ataque
* Maior controle de tráfego
* Melhor isolamento entre camadas
* Arquitetura alinhada a boas práticas cloud

### Negativas

* Maior complexidade de configuração inicial
* Necessidade de manutenção contínua de regras de firewall

---

## Conclusão

A adoção de uma política rigorosa de **segurança de rede em OCI** garante que o L2SLedger opere com isolamento adequado, menor exposição externa e alto nível de proteção, mesmo em um ambiente baseado em VMs e containers Docker.
